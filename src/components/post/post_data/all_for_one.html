I have been involved in conversations recently about mono repos and it has brought up a lot of questions.

What are the advantages? What are the drawbacks? Push to master means it is ready for prod? Why the assumption on one branch when multiple people are committing to the same branch? What if you want to stagger the changes being pushed? How do you deal with the communication of that with the team? How extensive is the testing on each commit?

You break something in one UI project, but the other changes are okay. If you use a mono repo and one branch, you now have to roll back everything. Please do not fall into the naive assumption that no broken code will go out.

You can resolve a lot with separate branches for each environment. Push to each branch and environment as it is ready. Build on the branch. Have to ability to pick, build and deploy on commits. Communicate what is going out and when. You are a team.
